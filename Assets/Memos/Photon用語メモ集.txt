Photon用語メモ集


[Networked]
全員で共有したい情報 本人がこの箱の中身を書き換えると、インターネットを通じて全員の画面にある同じ名前の箱の中身が自動的に書き換わります。
例えば  [Networked] private int hp { get; set; } という変数をA、B、Cというキャラクターがもっているとします。
Aのhpが減少したとき、もちろんA視点ではAのhpは減少しますが、そのときB、C視点からみたAのhpも減少します。

FixedUpdateNetwork (FUN)
自分にだけ反映: 自分自身であるかを確かめる条件式 if (HasStateAuthority) を書くことで、「自分のキャラの移動計算」を自分だけのPCで行います。
相手にも反映: 自分のPCで計算した結果（座標や[Networked]変数の値）がFusionによって相手に送られます。
相手側では「あ、Aさんの座標が更新されたな」と受け取るだけなので、相手のFUNで再計算する必要はありません。
つまり、A,Bというキャラクターがいた場合、Aが物理演算で移動したのをBも観測できます。
そのとき、Bさんの画面では「移動処理」自体は動いていません。
要するに、BさんのPCは、Aさんの FixedUpdateNetwork（移動ロジック）を実行しているわけではありません（HasStateAuthority で弾かれるため）。
Bさんの画面には、AさんのPCが計算した「最終的な場所」だけが届き、そこにキャラが配置されます。
ちなみに、NetworkRigidbodyも必要です。ただし、見た目は相手の画面に届けてくれません。そのときはRenderを使うなどする必要があります。
あくまで「物体としての位置と回転」を運ぶためのものだからです。(大きさの変化はFUNでは届けることができない)

Render
ここでは主に描画処理を記述します。これは相手の画面にも反映させることができます。逆に物理的な処理はここでは行うことができません。
アニメーションのパラメーター設定 (SetBool, SetTrigger)、向きや大きさの変更 (localScale)、エフェクト（パーティクル）の生成
SE（効果音）の再生、UI（名前や HP バー）の更新　これらのものはRenderで行います。
if (HasStateAuthority) を使用することで、先行反映ができます。選考反映とは、ラグを感じさせないよう、入力した瞬間に見た目を変えることを言います。
逆に、相手には同期反映になります。インターネット越しに届いた正しい状態を表示します。

OnInput
通常、UnityではUpdate内で Input.GetAxis などを直接使いますが、マルチプレイではそれだと不十分です。
なぜなら、「自分の入力」を「サーバーや他の人」に送らなければならないからです。
OnInput は、バラバラな入力情報（キー、マウス、ボタン）を一つの 「箱（構造体）」 に詰め込んで、Fusionに手渡す役割を持っています。
ちなみに、Renderで選考反映させるために、移動入力処理を実装し、OnInputにも移動入力処理を実装することがありますが、
Renderの方は自分だけの画面を最速で動かすためで、OnInputはネットワーク上の公式な記録のための入力です。そのため、少しだけラグが発生します。

ネットワーク系のメソッドで呼び出すか、普通のメソッドで呼び出すかの判断基準
この処理の結果が、隣で遊んでいる友達の画面でも『全く同じ位置・状態』になっていないとゲームが壊れるか？で判断します。
以下は普通のメソッドで呼び出すべき処理の例です。
1. 自分の入力（Input）の検知
2. UIの更新（HPバー、メニューなど）
3. カメラの追従
4. ローカルなエフェクト（パーティクル、音）

HasInputAuthority: 「このキャラをキーボードで動かしている人（操作者）」
HasStateAuthority: 「このキャラのHPや座標の正解を決める人（管理者）」

if(HasStateAuthority)が必要な理由
その通り、HasStateAuthority は「自分（そのオブジェクトの操作権・管理権を持っている人）」という意味で間違いありません。
なぜこれが必要かというと、Fusionにおいて [Networked] 変数（SyncHPなど）を書き換えられるのは、管理権限を持っているプレイヤーだけ というルールがあるからです。

もし if がなかったらどうなる？
マルチプレイ中、あなたの画面には「自分のキャラ」と「他人のキャラ」が映っていますよね。
自分のキャラ: あなたが StateAuthority（管理権限）を持っています。
他人のキャラ: 他の誰かが StateAuthority を持っています。
もし if を書かずに SyncHP = ... を実行しようとすると、あなたのコンピュータが「よし、他人のキャラのHP同期変数も書き換えてやろう！」と命令してしまいます。
するとFusionは、「権限がないのに同期変数をいじろうとしたな！」と判断して、エラー（Access Violation）を出したり、書き換えを無視したりします。
